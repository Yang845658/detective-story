# 📚 互動小說劇本編寫指南

> 基於搜索結果和最佳實踐整理

---

## 🎯 核心原則

### 1. 公平遊戲原則 (Fair Play)
**所有線索必須提前展示給玩家**

✅ 正確做法:
- 線索在對話中自然透露
- 每個結論都有證據支持
- 玩家可以通過邏輯推理得出真相

❌ 錯誤做法:
- 結局突然出現新線索
- 動機毫無預兆
- 強行解釋

### 2. 契訶夫之槍 (Chekhov's Gun)
**第一幕出現的槍，第三幕必須發射**

- 每個角色都應該有明確目的
- 每個線索都應該有作用
- 不要添加無關角色/線索

### 3. 紅鯡魚原則 (Red Herring)
**誤導必須合理**

- 誤導線索要有合理解釋
- 不能純粹為了誤導而誤導
- 真相應該比誤導更合理

## 📋 劇本結構模板

### 三幕結構

```
第一幕：鋪墊 (25%)
├─ 介紹主角和背景
├─ 發生案件
├─ 介紹嫌疑人 (2-3 人)
└─ 展示初步線索

第二幕：調查 (50%)
├─ 勘察現場 (收集線索 A)
├─ 詢問嫌疑人 1 (線索 B + 矛盾 1)
├─ 詢問嫌疑人 2 (線索 C + 矛盾 2)
├─ 發現關鍵證據
└─ 排除一個嫌疑人

第三幕：真相 (25%)
├─ 最終對質
├─ 揭示真相 (基於已有線索)
├─ 解釋動機 (前面有伏筆)
└─ 結局
```

## 🔍 推理遊戲設計要點

### 1. 線索設計

**三層線索系統:**

```
表層線索 (明顯)
├─ 所有人都能看到
├─ 指向多個嫌疑人
└─ 用於誤導

中層線索 (需要詢問)
├─ 通過對話獲得
├─ 排除部分嫌疑人
└─ 縮小範圍

核心線索 (關鍵)
├─ 通過推理發現
├─ 指向唯一真相
└─ 解釋所有矛盾
```

### 2. 嫌疑人設計

**每個嫌疑人需要:**
- ✅ 明確的動機 (提前展示)
- ✅ 作案機會 (時間線)
- ✅ 獨特性格 (對話風格)
- ✅ 隱藏秘密 (不一定是兇手)
- ❌ 不要添加無關人物

### 3. 時間線設計

```
必須清晰的時間軸:
18:00 - A 最後確認物品 (證人)
19:00 - B 進入現場 (嫌疑人 1)
20:00 - C 離開現場 (嫌疑人 2)
21:00 - 警報響起 (案發)
22:00 - 偵探抵達 (調查開始)

每個時間點都需要:
- 誰在場
- 誰不在場
- 誰能證明
```

## ❌ 常見錯誤

### 錯誤 1: 突然出現的線索
```
❌ 錯誤:
結局：「因為他欠了 50 萬賭債」
(前面從未提及賭債)

✅ 正確:
第 1 章：「張偉最近總接到催債電話」
第 2 章：「張偉的銀行賬戶被凍結了」
第 3 章：「張偉承認欠了賭債」
結局：「因為他欠了 50 萬賭債」
```

### 錯誤 2: 無關角色
```
❌ 錯誤:
王老爺子只在結局出現
沒有台詞、沒有線索、沒有作用

✅ 正確:
要麼刪除这个角色
要麼給他：
- 第 1 章登場
- 2-3 次對話
- 獨特線索
- 合理動機
```

### 錯誤 3: 線索顯示過快
```
❌ 錯誤:
剛進入場景就顯示所有線索
對話變得沒有意義

✅ 正確:
對話進行到某處 → 獲得線索
詢問特定問題 → 獲得線索
檢查特定物品 → 獲得線索
```

### 錯誤 4: 場景沒有互動
```
❌ 錯誤:
場景只是背景圖片
沒有任何可點擊/調查的元素

✅ 正確:
每個場景都有:
- 可調查的物品 (2-3 個)
- 可詢問的人物
- 可收集的線索
```

## 📖 逆轉裁判式設計

### 核心循環
```
調查階段:
├─ 勘察現場 (點擊調查)
├─ 收集證據 (放入法庭記錄)
└─ 詢問證人 (獲得證詞)

法庭階段:
├─ 聽取證詞
├─ 發現矛盾 (按壓)
├─ 出示證據 (反駁)
└─ 揭穿謊言
```

### 證詞設計
```
證詞格式:
「案發時我在房間裡看書」

矛盾點:
- 書是乾淨的 (如果真在看，應該有灰塵)
- 燈是關的 (怎么看書？)
- 其他人證明他不在房間

證據:
- 指紋報告
- 照片
- 證人證詞
```

## 🎯 實用檢查清單

### 劇本完成後檢查

**角色檢查:**
- [ ] 每個角色都有明確目的
- [ ] 每個角色都有獨特性格
- [ ] 沒有無關角色
- [ ] 姓名一致 (無拼寫錯誤)

**線索檢查:**
- [ ] 所有線索都提前展示
- [ ] 每個結論都有證據支持
- [ ] 沒有突然出現的線索
- [ ] 線索獲取時機合理

**邏輯檢查:**
- [ ] 時間線連貫
- [ ] 證詞無自相矛盾 (除非設計)
- [ ] 動機合理且有伏筆
- [ ] 真相能解釋所有疑點

**結局檢查:**
- [ ] 好結局基於證據
- [ ] 壞結局有說服力
- [ ] 沒有強行解釋
- [ ] 玩家能理解為什麼輸/贏

## 📝 修改建議

### 針對「午夜美術館」的問題

**問題 1: 場景空白**
```
修改:
每個場景添加可點擊元素
- 展櫃: 檢查玻璃碎片
- 警報器: 檢查記錄
- 地板: 檢查腳印
```

**問題 2: 王老爺子多餘**
```
修改方案 A: 刪除
- 只保留 2 名嫌疑人
- 張偉 (保安) vs 李娜 (策展人)

修改方案 B: 添加完整劇情
- 第 1 章登場 (捐贈者身份)
- 第 2 章詢問 (最後见到鑽石的人)
- 第 3 章提供關鍵線索
- 結局有合理動機
```

**問題 3: 線索顯示過快**
```
修改:
改為對話中逐步獲得
- 說完第 1 句 → 獲得線索 A
- 詢問特定問題 → 獲得線索 B
- 檢查特定物品 → 獲得線索 C
```

**問題 4: 賭債突然出現**
```
修改:
提前鋪墊
- 第 1 章：張偉接電話「我、我真的沒錢了...」
- 第 2 章：李娜說「張偉最近財務狀況不好」
- 第 3 章：發現賭債文件
- 結局：揭露賭債動機
```
